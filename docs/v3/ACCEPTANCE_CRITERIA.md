# Acceptance Criteria

## Content and credibility

- No simulated staff, vehicle, job, review, or operational proof is presented as real.
- All visible contact data and trust claims are verified configuration.
- Core services, inclusions, exclusions, starting prices, and quote factors are clear.
- Legal/policy draft status is documented.

## Conversion

- Every primary Request Estimate CTA reaches the public Marble Quote Portal.
- Call/email actions work where configured.
- Portal-unavailable fallback is transparent and usable.
- Mobile conversion controls do not obstruct content or accessibility.

## Routes and SEO

- Every required route exists and returns a successful response.
- Each route has unique title, description, canonical, H1, and useful content.
- Sitemap contains canonical routes; robots references sitemap.
- Structured data validates and contains verified facts only.
- No thin duplicate location pages.

## Accessibility

- Keyboard-only navigation works.
- Focus is visible and logical.
- Mobile menu state/focus is correct.
- Automated critical accessibility violations: zero.
- Contrast, reduced motion, headings, landmarks, labels, and alt text pass review.

## Performance and quality

- Production build succeeds cleanly.
- Internal link check has zero broken links.
- Mobile Lighthouse targets are met or deviations documented and approved.
- No console errors on primary pages.
- Chrome, Edge, Safari/iOS, and Android viewport checks pass.
- Custom 404 works.

## Deployment

- Cloudflare preview reviewed before production.
- Apex canonical and `www` redirect verified.
- Security headers verified.
- Verification report and release manifest are complete.
